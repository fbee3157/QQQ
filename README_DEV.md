# 开发文档

## 项目架构

```
├── live_trader.py      # 核心交易引擎
├── trader_web.py       # Web 仪表盘
├── watchdog.py         # 进程守护
├── update_gist.py      # Gist 同步（可选）
├── config.py           # 配置管理
├── indicators.py       # 技术指标库
├── state.json          # 实时状态（自动生成）
├── records/            # 交易记录（自动生成）
└── .env                # 密钥配置（不入库）
```

## 核心改进说明

### 1. 参数体系优化
- 统一参数配置到 `CONFIG` 字典
- 与 `config.md` 保持一致
- 所有策略参数集中管理

### 2. 信号生成优化
#### 信号 1：趋势突破（顺势）
- 价格突破前 N 根 K 线高/低点
- 4 层过滤：
  - 趋势过滤（SMA20）
  - 动量过滤（3 周期变化率）
  - 量能过滤（成交量倍数）
  - K 线实体过滤（防止十字星）

#### 信号 2：衰竭反转（逆势）
- 从日内高/低点的大幅回落/反弹
- 需要 K 线实体确认
- 每天多次，防止频繁交易

### 3. 止盈逻辑优化
三层止盈机制（按优先级）：
1. **止损**：亏损 25% 立即平仓
2. **超时止损**：持仓 15 分钟平仓
3. **部分止盈**：盈利 100% 平仓一半锁定利润
4. **回撤止盈**：从最高盈利回撤 30% 全部平仓

### 4. 日亏损管理
- 追踪日累计收益率
- 日亏损达 5% 停止交易
- 每天自动重置

### 5. 技术指标增强
`indicators.py` 新增指标：
- `ema()`：指数移动平均线
- `rsi()`：相对强弱指标
- `atr()`：平均真实波幅
- `std_dev()`：标准差
- `momentum()`：动量绝对值

### 6. 错误处理改进
- 所有 API 调用添加异常捕获
- 连接失败自动重试
- 记录详细错误日志

### 7. Web 仪表盘美化
- 现代化 UI 设计（卡片布局）
- 实时状态指示器
- 交易记录表格展示
- 收益率彩色标记（正负）

### 8. 日志优化
- 统一日志格式
- 关键操作添加 ✓/✗ 符号
- 按日期输出（便于定位问题）

## 文件说明

| 文件 | 说明 |
|------|------|
| `live_trader.py` | 核心交易引擎，包含信号生成、下单、持仓管理 |
| `trader_web.py` | Flask Web 服务，提供仪表盘和 API |
| `watchdog.py` | 进程守护，自动重启子进程 |
| `update_gist.py` | GitHub Gist 同步脚本 |
| `config.py` | 配置加载、环境变量管理 |

| `state.json` | 实时运行状态（自动生成） |
| `records/*.json` | 每笔交易的详细记录 |

## 调优建议

### 信号灵敏度调整
- 降低 `vol_mult` 值：增加交易机会（1.2 → 0.8）
- 增加 `min_body` 值：过滤更多噪音（0.0003 → 0.0005）
- 调整 `lookback` 值：3（敏感）→ 7（保守）

### 风险管理调整
- 调整 `sl`：止损率（0.20 → 0.30）
- 调整 `tp_partial_pct`：部分止盈点（0.80 → 1.20）
- 调整 `tp_trail_drop`：回撤止盈幅度（0.25 → 0.40）

### 交易窗口调整
- `start_time`：开市时间（推荐 09:35 避免开盘噪音）
- `end_time`：收盘时间（推荐 15:50 避免尾盘波动）

## 故障排查

### 问题：连接失败
```
solution：检查 .env 文件中的 API 密钥是否正确
```

### 问题：信号无输出
```
solution：
1. 检查 K 线数据：state.json 中 candle_count 是否 > 0
2. 检查交易窗口：当前时间是否在 09:35-15:50（美东时间）
3. 检查持仓阻塞：是否已有开仓位置
```

### 问题：期权下单失败
```
solution：
1. 确保账户已开通美股期权交易权限
2. 确认行权价是否符合市场（±$2 范围内）
3. 检查账户资金是否充足
```
