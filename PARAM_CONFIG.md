# 参数配置使用指南 (v6.2+)

## 📊 功能概述

v6.2 引入了**可视化参数配置系统**，允许在 Web 仪表盘中实时调整和保存交易策略参数，无需重启交易引擎。

### 核心特性
- ✅ **Web UI 参数表单** - 直观的分类参数管理界面
- ✅ **参数实时验证** - 自动检查参数有效性
- ✅ **参数热加载** - 修改后自动应用到交易引擎
- ✅ **参数持久化** - 保存到 `config_params.json`
- ✅ **参数重置** - 一键恢复默认值
- ✅ **参数分类** - 6 大类别便于管理

---

## 🚀 快速开始

### 1. 访问参数配置页面

启动系统后，打开仪表盘：
```
http://127.0.0.1:8080
```

点击顶部导航栏的 **⚙️ 参数配置** 按钮

### 2. 调整参数

#### 方式 A：编辑表单
```
1. 在参数输入框中修改数值
2. 按 Enter 或点击其他位置
3. 系统自动保存并验证
4. 成功时显示绿色提示
```

#### 方式 B：编辑 JSON 文件
```bash
# 直接编辑参数文件（需停止交易引擎）
vim config_params.json
```

### 3. 验证参数效果

通过仪表盘观察：
- 交易信号更新
- 交易次数变化
- 收益率变化

## 📋 参数分类说明

### 1️⃣ 基础配置 (Basic Config)

| 参数 | 说明 | 建议值 | 范围 |
|------|------|--------|------|
| `symbol` | 交易标的 | QQQ.US | QQQ.US / SPY.US / QQQA.US |
| `capital` | 账户资金 (USD) | 100000 | 1,000 - 10,000,000 |
| `check_interval` | 检测间隔 (秒) | 20 | 5 - 60 |

**使用建议**：
- symbol 改为 SPY.US 后自动适配 SPY 交易
- capital 影响仓位计算，更新时需确保精准
- check_interval 越小延迟越低，但 CPU 占用越高

### 2️⃣ 风险管理 (Risk Management)

| 参数 | 说明 | 建议值 | 范围 | 用途 |
|------|------|--------|------|------|
| `sl` | 止损率 | 0.25 | 0.10-0.50 | 亏损达此比例时平仓 |
| `tp` | 止盈率 | 0.30 | 0.10-1.00 | 兼容旧逻辑 |
| `tp_partial_pct` | 部分止盈点 | 1.00 | 0.30-2.00 | 盈利达此比例时平仓一半 |
| `tp_trail_drop` | 回撤止盈 | 0.30 | 0.10-0.50 | 从最高盈利回撤此比例时全仓 |
| `max_trades` | 日最大交易笔数 | 8 | 1-20 | 日交易限制 |
| `daily_limit` | 日亏损限制 | 0.05 | 0.01-0.20 | 日亏损达此比例时停止 |

**调优场景**：

**激进型**（追求高频交易）
```json
"sl": 0.20,
"tp_partial_pct": 0.50,
"tp_trail_drop": 0.15,
"max_trades": 15,
"daily_limit": 0.10
```

**保守型**（追求稳定收益）
```json
"sl": 0.30,
"tp_partial_pct": 1.50,
"tp_trail_drop": 0.40,
"max_trades": 5,
"daily_limit": 0.03
```

### 3️⃣ 信号过滤 (Signal Filter)

| 参数 | 说明 | 建议值 | 范围 | 影响 |
|------|------|--------|------|------|
| `lookback` | 突破窗口 (K线数) | 5 | 3-10 | ⬇ 灵敏 / ⬆ 保守 |
| `vol_mult` | 成交量倍数 | 0.8 | 0.5-2.0 | ⬇ 放宽 / ⬆ 严格 |
| `min_body` | K线实体比例 | 0.0003 | 0.0001-0.01 | ⬇ 放宽 / ⬆ 严格 |
| `max_gap` | 最大跳空 | 0.002 | 0.001-0.01 | ⬇ 允许更大跳空 |

**调优建议**：

**信号太少** 👇
```
降低 lookback: 5 → 3
降低 vol_mult: 0.8 → 0.6
降低 min_body: 0.0003 → 0.0001
```

**假信号太多** 👆
```
提高 lookback: 5 → 7
提高 vol_mult: 0.8 → 1.2
提高 min_body: 0.0003 → 0.0005
```

### 4️⃣ 衰竭反转 (Exhaustion Reversal)

| 参数 | 说明 | 建议值 | 范围 |
|------|------|--------|------|
| `reversal_drop` | 高点跌幅 | 0.002 | 0.001-0.01 |
| `reversal_bounce` | 反弹实体 | 0.001 | 0.0001-0.01 |

**使用场景**：
- 从日内最高点回落 ≥ 0.2% 时触发做多信号
- 从日内最低点反弹 ≥ 0.2% 时触发做空信号
- 降低阈值 → 更多反弹交易 → 容易被套
- 提高阈值 → 等待更大反弹 → 错过机会

### 5️⃣ 交易窗口 (Trading Window)

| 参数 | 说明 | 建议值 | 格式 |
|------|------|--------|------|
| `start_time` | 交易开始时间 | 09:35 | HH:MM (美东) |
| `end_time` | 交易结束时间 | 15:50 | HH:MM (美东) |

**时间选择原理**：
- **09:30-09:35**：开盘前 5 分钟，波动大噪音多 → 避开
- **09:35-15:50**：正常交易时段，流动性好 → 主要交易时间
- **15:50-16:00**：收盘前 10 分钟，波动剧烈 → 避开

**自定义示例**：
```json
"start_time": "10:00",  // 避开开盘急速涨跌
"end_time": "15:30"     // 提前收盘避免风险
```

### 6️⃣ 期权参数 (Option Parameters)

| 参数 | 说明 | 建议值 | 注意 |
|------|------|--------|------|
| `option_offset` | 行权价偏移 (±$) | 2.0 | OTM 虚值期权 |
| `min_contracts` | 最小张数 | 10 | 1张=100股 |
| `contract_multiplier` | 合约乘数 | 100 | 标准期权 |
| `pos_pct` | 仓位百分比 | 0.02 | 占总资金比例 |

**说明**：
- option_offset: 选择 ±$2 虚值期权（既防护又降低成本）
- min_contracts: 最小 10 张（较少会被滑点影响）
- pos_pct: 2% 较为保守，激进可改为 5%

## 🔄 参数热加载机制

### 工作原理

```
修改参数 (Web UI)
       ↓
保存到 config_params.json
       ↓
交易引擎每 30 秒扫描一次
       ↓
检测到参数变化
       ↓
自动更新内存中的 CONFIG
       ↓
新参数即刻生效
```

### 生效时间

| 操作 | 生效时间 | 说明 |
|------|---------|------|
| 调整参数 | 保存后 30 秒内 | 下次参数检查时生效 |
| 下单条件 | 即刻 | 新信号立即按新参数判断 |
| 已持仓 | 持续应用 | 回撤止盈等条件重新计算 |

### 示例场景

**场景**：实时调整止损

```
1. 14:30 - 交易引擎运行中，参数 sl=0.25
2. 14:35 - 发现策略损失过大，Web UI 改为 sl=0.15
3. 14:40 - 参数热加载成功，新信号使用 sl=0.15
4. 14:41 - 已持仓按新止损规则评估，可能立即平仓
```


## 🛠️ API 参考

### 获取所有参数
```bash
curl http://127.0.0.1:8080/api/params
```

**响应**：
```json
{
  "symbol": "QQQ.US",
  "capital": 100000,
  "sl": 0.25,
  ...
}
```

### 更新参数
```bash
curl -X POST http://127.0.0.1:8080/api/update_param \
  -H "Content-Type: application/json" \
  -d '{"sl": 0.20, "max_trades": 10}'
```

**响应**：
```json
{
  "success": true,
  "updated": ["sl", "max_trades"]
}
```

### 重置参数
```bash
curl -X POST http://127.0.0.1:8080/api/reset_params
```

**响应**：
```json
{
  "success": true,
  "message": "参数已重置"
}
```

### 获取参数元数据
```bash
curl http://127.0.0.1:8080/api/param_info
```

## ⚠️ 注意事项

### 不要修改的参数
- `contract_multiplier` - 固定为 100（美股期权标准）
- `symbol` - 改动需重启引擎重新初始化

### 验证规则
- **数值范围**：超出范围的输入会被拒绝
- **类型检查**：整数/浮点/字符串格式自动验证
- **依赖关系**：某些参数组合可能不合理（如 tp < sl）

### 参数对性能的影响

| 参数调整 | 交易频率 | CPU 占用 | 风险 | 建议 |
|---------|---------|---------|------|------|
| ⬇ lookback | ⬆ | - | ⬆ | 平衡 |
| ⬇ vol_mult | ⬆ | - | ⬆ | 逐步降低 |
| ⬆ check_interval | - | ⬇ | ⬆ | 不低于 10 |
| ⬇ min_contracts | ⬆ | - | ⬆ | 不低于 5 |

## 🎓 参数优化工作流

### Step 1: 基准测试
```
1. 使用默认参数运行 1 周
2. 记录胜率、平均收益、最大回撤
3. 建立基准数据
```

### Step 2: 逐个调优
```
1. 选择一个参数（如 lookback）
2. 修改为 -20% 或 +20%
3. 运行 2-3 天观察效果
4. 对比基准数据
```

### Step 3: 组合优化
```
1. 根据 Step 2 结果选择最优参数
2. 固定这些参数
3. 调整下一组参数
4. 重复直到收敛
```

### Step 4: 验证稳定性
```
1. 使用最优参数运行 2 周
2. 验证相对稳定的胜率和收益
3. 部署生产环境
```

## 🐛 故障排查

### 参数无法保存
```
问题：点击保存后页面没反应
原因：1. Web 服务未运行
     2. JSON 格式错误
     3. 参数超出范围
解决：1. 检查服务状态：ps aux | grep trader_web.py
     2. 查看浏览器控制台（F12）的错误信息
     3. 检查参数范围是否合规
```

### 参数未被应用
```
问题：修改了参数但交易策略没变化
原因：1. 交易引擎还未加载新参数（30 秒内）
     2. 参数文件保存失败
解决：1. 等待 30 秒后查看日志
     2. 检查 config_params.json 文件权限
     3. 查看 live_trader.py 日志确认热加载状态
```

### 参数验证失败
```
问题：输入值被拒绝
原因：1. 数值超出允许范围
     2. 数据类型错误（如字符串输入数字字段）
解决：检查参数表中的最小/最大值范围
```
